# Java Architecture - Core Concepts

## üß± Java Platform Overview

Java is a platform-independent, object-oriented programming language. It follows this architecture:


## ‚öôÔ∏è Components of Java Architecture

| Component | Description |
|----------|-------------|
| **JDK** | Java Development Kit ‚Äì includes tools like `javac`, `javadoc`, `javap`, and the JRE |
| **JRE** | Java Runtime Environment ‚Äì JVM + core libraries |
| **JVM** | Java Virtual Machine ‚Äì runs `.class` files on your platform |
| **Bytecode** | Intermediate code generated by `javac` |
| **Java** | Language used to write platform-independent code |

---

## üéØ Why Java is Platform Independent?

- Java code is compiled to **bytecode**, not machine code.
- Bytecode runs on **JVM**, which is platform-specific.
- So you write once, and run anywhere JVM is available.

---

## üî• Real-World Analogy

| Real-world | Java Equivalent |
|------------|-----------------|
| Script in English | Java Code |
| Translated to Spanish | Bytecode |
| Interpreter (Spanish speaker) | JVM |
| Local culture rules | OS (Windows/Linux) |

---

## üß† FAANG-Style Interview Q&A

### ‚ùì Why is Java platform-independent?
‚úÖ Because Java code is compiled to bytecode, and bytecode runs on any machine with a JVM.

### ‚ùì What is the role of the JVM?
‚úÖ JVM loads, verifies, and executes Java bytecode on the host system.

### ‚ùì What does JDK include?
‚úÖ JDK = JRE + development tools like compiler, debugger, etc.

# üìò Module 03: Control Flow in Java (Branching & Loops)

Control flow lets your program **make decisions** (`if`, `switch`) and **repeat actions** (`for`, `while`, etc.).

---

## 1Ô∏è‚É£ Conditional Branching

### üî∏ `if`, `else if`, `else`

Real-time use: Swiggy delivery status

> If distance < 2km ‚Üí show "Fast Delivery"
> Else if distance < 5km ‚Üí "Standard Delivery"
> Else ‚Üí "Slow Delivery"

```java
int distance = 4;

if (distance < 2) {
    System.out.println("Fast Delivery");
} else if (distance < 5) {
    System.out.println("Standard Delivery");
} else {
    System.out.println("Slow Delivery");
}
```

---

### üî∏ `switch` Statement

Real-time use: Days of the week in a calendar app.

```java
int day = 3;

switch (day) {
    case 1: System.out.println("Sunday"); break;
    case 2: System.out.println("Monday"); break;
    case 3: System.out.println("Tuesday"); break;
    default: System.out.println("Invalid day");
}
```

> ‚úÖ Use `switch` when checking a variable against multiple constant values (especially `int`, `char`, `String` in Java 7+).

> Always run at least once (example: user input menu)


# üß± Module 04: Object-Oriented Programming (OOP) Basics in Java

This module introduces the **core concepts of OOP** ‚Äî the foundation for scalable, maintainable Java applications. These concepts are frequently asked in FAANG interviews.

---

## üì¶ 1Ô∏è‚É£ Class & Object

### ‚û§ What is a Class?

A **class** is a blueprint for creating objects (instances). It defines **state (variables)** and **behavior (methods)**.

> üß† Real-world Analogy: A class is like the blueprint of a car.

```java
class Car {
    String color;
    int speed;

    void drive() {
        System.out.println("Driving at " + speed + " km/hr");
    }
}
```

### ‚û§ What is an Object?

An **object** is an instance of a class.

```java
Car myCar = new Car();
myCar.color = "Red";
myCar.speed = 80;
myCar.drive();
```

---

## üö™ 2Ô∏è‚É£ Constructors

A **constructor** is a special method called when an object is created. Its job is to **initialize object values**.

```java
class User {
    String name;
    int age;

    // Constructor
    User(String n, int a) {
        name = n;
        age = a;
    }
}

User u = new User("Udhay", 25);
```

---

## üîÅ 3Ô∏è‚É£ Constructor Overloading & Chaining

### ‚û§ Constructor Overloading

Creating multiple constructors with different parameter lists.

```java
class Book {
    String title;
    int pages;

    Book(String t) {
        title = t;
        pages = 100;
    }

    Book(String t, int p) {
        title = t;
        pages = p;
    }
}
```

### ‚û§ Constructor Chaining using `this()`

You can call **one constructor from another** using `this()`.

> üß† Real-life: If quantity isn't given in a Swiggy order, assume it's 1.

```java
class Order {
    String item;
    int quantity;

    Order(String item) {
        this(item, 1); // calls the second constructor
    }

    Order(String item, int qty) {
        this.item = item;
        this.quantity = qty;
    }
}

Order o1 = new Order("Pizza");        // quantity = 1
Order o2 = new Order("Burger", 2);     // quantity = 2
```

---

## üîê 4Ô∏è‚É£ Access Modifiers

Access modifiers control **who can access a class's members**.

| Modifier    | Access Level                 |
| ----------- | ---------------------------- |
| `public`    | Accessible from anywhere     |
| `private`   | Only within the class        |
| `protected` | Same package or subclass     |
| (default)   | Only within the same package |

```java
class BankAccount {
    private double balance;

    public void deposit(double amount) {
        balance += amount;
    }

    public void showBalance() {
        System.out.println("Balance: " + balance);
    }
}
```
